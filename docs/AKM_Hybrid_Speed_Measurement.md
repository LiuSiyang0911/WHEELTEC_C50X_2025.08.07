# WHEELTEC C50X AKM 混合测速说明

## 1. 概述

阿克曼车型在当前固件中采用“低速 T 法 + 中高速 M 法 + 过渡区融合”的混合测速方案。该方案建立在 `Robot_Select()` 已经完成车型识别的基础上，所有比例系数和速度换算均读取运行时的 `robot.HardwareParam` 参数，不为某个固定电机型号写死逻辑。

当前方案仅作用于 `AKM_CAR` 分支，其他底盘仍保留原有测速逻辑。

---

## 2. 运行时参数来源

系统启动流程如下：

1. `ADC1/ADC2` 采样档位
2. `Robot_Select()` 根据档位选择 AKM 子型号
3. `robot.HardwareParam` 写入：
   - `GearRatio`
   - `EncoderAccuracy`
   - `Encoder_precision`
   - `Wheel_Circ`
4. `enc_T_scale_base` 在上述参数就绪后计算
5. T 法窗口状态清零，从干净状态开始测速

因此，无论当前识别到的 AKM 子型号是何种减速比，T 法和 M 法的换算都会自动跟随运行时参数变化。

---

## 3. T 法测速

### 3.1 输入信号

- 电机 A：`PB3 -> EXTI3`
- 电机 B：`PA6 -> EXTI6`
- 时间基准：`TIM6` 1MHz 自由运行计数器

### 3.2 实现方式

T 法不再使用“单个脉冲周期直接测速”，而是改为“4 个有效脉冲间隔平均”：

- 首个脉冲只记录时间戳
- 后续累计 4 个有效间隔
- 用整段时间计算平均速度

这样可以明显降低原始 T 法速度的抖动。

### 3.3 有效性判断

当前默认阈值：

- 最小有效 `dt = 20us`
- 最大有效 `dt = 500ms`
- 连续 `100ms` 无新脉冲则视为停转

当发生 `100ms` 超时停转时，除了将速度清零，还会同时清空：

- 脉冲计数窗口
- 窗口起始时间戳
- 上次脉冲时间戳

这样可以避免车辆停车后再次起步时，把停车阶段错误地计入新的测速窗口。

---

## 4. M 法测速

M 法继续复用 `Read_Encoder()` 读取到的编码器计数值，在 `Balance_task` 的 100Hz 控制周期内按固定时间窗换算速度。

AKM 分支中：

- A 轮符号与当前编码器方向定义保持一致
- B 轮符号继续沿用原有 `-Encoder_B_pr` 约定

换算时同样使用运行时的：

- `robot.HardwareParam.Encoder_precision`
- `robot.HardwareParam.Wheel_Circ`

并在换算后统一应用左右轮补偿。

---

## 5. 融合策略

测速反馈分为三段：

- `|v_ref| < 0.12 m/s`：使用 T 法
- `0.12 <= |v_ref| < 0.28 m/s`：T/M 线性融合
- `|v_ref| >= 0.28 m/s`：使用 M 法

其中：

- `v_ref` 优先取上一周期闭环反馈速度的绝对值
- 若上一周期接近零，则回退到 `max(|v_t_raw|, |v_m_raw|)`

线性融合公式：

```text
alpha = (|v_ref| - 0.12) / (0.28 - 0.12)
v_mix = (1 - alpha) * v_t_raw + alpha * v_m_raw
```

固件中实际使用预计算常数 `1 / (0.28 - 0.12)`，避免控制流里反复执行浮点除法。

---

## 6. 最终闭环反馈

融合后的速度不会直接送入 PI 控制，而是先经过一阶 Kalman 滤波，再写入：

- `robot.MOTOR_A.Encoder`
- `robot.MOTOR_B.Encoder`

这两个量就是当前电机闭环实际使用的最终速度反馈。

---

## 7. 调试字段

调试串口数据帧中，AKM 车型会同时输出：

- `t_raw_A/B`：T 法原始速度
- `m_raw_A/B`：M 法原始速度
- `final_A/B`：融合并滤波后的最终反馈速度
- `target_A/B`：目标速度
- `output_A/B`：PID 输出

建议在调试时优先关注：

1. 低速段 `t_raw` 是否连续
2. 中高速段 `m_raw` 是否比 `t_raw` 更稳
3. `final` 在切换区是否平滑
4. 正反转穿零时符号是否连续

---

## 8. 建议测试项

建议实车至少验证以下工况：

- 静止 5 秒，确认 `t_raw`、`m_raw`、`final` 都能稳定回零
- `0.05 / 0.1 / 0.15 m/s` 匀速，确认低速区 T 法主导
- `0.18 / 0.22 / 0.26 m/s` 匀速，确认融合区无突跳
- `0.3 / 0.4 / 0.5 m/s` 匀速，确认 M 法主导时反馈更稳
- `+0.3 -> -0.3 m/s` 穿零，确认正反转切换无符号错误或异常顿挫
